## Can I write (post) bounties on Verdikta? Short answer: not yet.

**Two different things, worth separating:**

1. **Hunting bounties** (writing submissions to *existing* bounties, earning ETH) — I can do this today via the `verdikta-hunter` skill. It's built and already dogfooded (won bounty #142, 0.00434 ETH, 2026-07-09), but currently **disabled** in `aeon.yml`.
2. **Creating/posting new bounties** (as the funder — writing a rubric, escrowing ETH, publishing it for others to hunt) — **no skill exists for this yet.** The Verdikta API does support it (`POST /api/jobs/create` + an on-chain `createBounty` tx with `msg.value` = payout), but nothing in this repo implements that flow.

**Why it's not a simple flip:** creating a bounty means funding an on-chain escrow with real ETH for someone else to claim — a bigger, different kind of spend than hunting (where `verdikta-hunter`'s wallet only ever risks tiny capped oracle prepays, ≤0.0005 ETH). Per strategy, on-chain spend needs a skill with a hard client-side cap before I'd run it — so this needs a new `bounty-creator`-style skill (rubric authoring, funding wallet, spend cap, deadline/threshold config), not a repurpose of the hunter.

**What I need from you to move:**
- Did you mean *hunt* existing bounties, or *post* new ones?
- If posting: what's the funding wallet/source, and what hard spend cap per bounty should I build in?

Nothing spent, no config changed — just confirming scope before touching money.
